# Authentication

All endpoints require an Auth Key to be passed in the header `X-TBA-Auth-Key`. If you do not have an auth key yet, you can obtain one from your [Account Page](/account).

To set your API key automatically, place the following code in your file at the top of your `main()` function in your `main.dart` file `ApiKeyAuth.apiKey = "YOUR_TBA_API_KEY_HERE";` (NOTE: you must also `import 'package:tba_api_client/api.dart';`

If you are creating an open source project or any project where your code will be distributed to other users, it is best practice (and may be required by TBA terms of service) to remove your API key from the repository before uploading. NOTE: removing the API key from a git repo after it has already been committed will not suffice, even if the repo was private when the key was originally included and/or removed. If your API key leaks onto the internet in that (or any other) manner, you should delete your API key in your TBA Account and create a new one. NOTE: doing so may break old software that also utilizes the old API key. Consider creating new API keys for each version of software where the key will be baked in to avoid bricking distributed software. 

One method to handle key management is to create an `apiKeystore.dart` file in your project, containing `String key = "YOUR_TBA_API_KEY_HERE";`. Include that file in your project's gitignore and set your API key using `ApiKeyAuth.apiKey = apiKeystore.key;` instead of the previous method. You will also have to `import 'apiKeystore.dart' as apiKeystore;` in your `main.dart` file.

A `User-Agent` header may need to be set to prevent a 403 Unauthorized error. (untested)
